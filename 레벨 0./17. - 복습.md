#  17. REST API에서 HTTP 메서드들의 차이점을 설명해주세요.

# ✅ REST API — 모든 HTTP 메서드 핵심 정리

HTTP 메서드 = **서버에게 시키는 행동 명령어**
레스토랑 비유로 설명함

---

# 1️⃣ GET — “메뉴판 좀 보여주세요”

**조회 (Read)**

* 서버에서 **데이터 가져오기**
* URL로 전달
* 서버 상태 안 변함

---

# 2️⃣ POST — “이거 주문할게요”

**생성 (Create)**

* 서버에 **새 데이터 제출**
* Body에 데이터 전송
* 서버 상태 **변함**

---

# 3️⃣ PUT — “이 메뉴 그냥 통째로 바꿔주세요”

**전체 수정 (Update All)**

* 데이터를 **완전히 교체**
* 기존 데이터를 **전부 갈아끼움**
* 새로 보내는 데이터가 ‘전체 값’이어야 함

**비유**
👉 음식 통째로 새로 만들어서 가져다줌
기존 음식은 완전 삭제됨

---

# 4️⃣ PATCH — “이 부분만 살짝 고쳐주세요”

**부분 수정 (Partial Update)**

* 데이터의 **일부만 수정**
* 필요한 값만 보내도 됨

**비유**
👉 음식 전체를 바꾸는 게 아니라 토핑만 추가하는 느낌

---

# 5️⃣ DELETE — “이 메뉴 취소해주세요”

**삭제 (Delete)**

* 서버의 특정 데이터 삭제

**비유**
👉 서버에게 “이 주문은 없던 걸로요”라고 말하는 것

---

# 📌 전체 요약 표

| 메서드        | 기능    | 서버 변화 여부 | 비유        |
| ---------- | ----- | -------- | --------- |
| **GET**    | 조회    | 변화 없음    | 메뉴판 보기    |
| **POST**   | 생성    | 변화 있음    | 주문하기      |
| **PUT**    | 전체 수정 | 변화 있음    | 음식 통째로 교체 |
| **PATCH**  | 부분 수정 | 변화 있음    | 토핑만 수정    |
| **DELETE** | 삭제    | 변화 있음    | 주문 취소     |

---

# 📌 단 5줄로 외우기 (평생 기억용)

* **GET = 가져오기**
* **POST = 만들기**
* **PUT = 통째로 바꾸기**
* **PATCH = 일부만 고치기**
* **DELETE = 지우기**
