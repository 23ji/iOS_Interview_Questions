#  26. 동적 프로그래밍(Dynamic Programming)의 개념을 설명해주세요.

# 1️⃣ 핵심 개념

**동적 프로그래밍 (Dynamic Programming, DP)**
→ **큰 문제를 작은 문제로 나누고, 작은 문제의 결과를 저장해서 재사용하는 방식**

핵심 아이디어

```id="z30q9g"
같은 계산을 반복하지 않음
```

---

# 2️⃣ 왜 필요한가

재귀만 사용하면

같은 계산이 여러 번 반복됨

예

피보나치

```id="ymk4fa"
f(5)

→ f(4) + f(3)

→ f(3) 또 계산됨
```

중복 계산 많아짐

DP는

```id="g9q0wq"
한 번 계산한 값 저장
```

해서 다시 사용

---

# 3️⃣ 적용 조건

DP는 보통 두 조건 필요

| 조건       | 의미                  |
| -------- | ------------------- |
| 중복 부분 문제 | 같은 작은 문제가 반복됨       |
| 최적 부분 구조 | 작은 문제 해로 큰 문제 해결 가능 |

---

# 4️⃣ 구현 방식

## Top-Down

재귀 + 메모이제이션

```id="yixf9j"
필요할 때 계산하고 저장
```

---

## Bottom-Up

반복문으로 작은 값부터 채움

```id="nhk9lq"
아래부터 순서대로 계산
```

---

# 5️⃣ 시간 복잡도 개선 예

피보나치

기존 재귀

```id="0sj8g2"
O(2^n)
```

DP 사용

```id="t3m6rf"
O(n)
```

---

# 6️⃣ 면접 답변 요약

동적 프로그래밍은 큰 문제를 작은 문제로 나누고, 중복 계산되는 작은 문제의 결과를 저장하여 재사용하는 알고리즘 설계 방식입니다. 중복 부분 문제와 최적 부분 구조를 만족할 때 적용하며, 메모이제이션이나 반복문 기반 Bottom-Up 방식으로 구현합니다.

---

# 📌 핵심 요약

* **DP = 결과 저장 후 재사용**
* 같은 계산 반복 제거
* 조건: **중복 부분 문제 + 최적 부분 구조**
* 구현: **Top-Down / Bottom-Up**
* 시간 복잡도 크게 개선 가능
