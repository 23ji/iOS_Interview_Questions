#  23. 알고리즘의 시간 복잡도와 공간 복잡도의 개념, 그리고 빅오 표기법에 대해 설명해주세요.

### 1️⃣ 시간 복잡도 (Time Complexity)

* 입력 크기(n)가 증가할 때
* **실행 시간이 어떻게 증가하는지**를 나타내는 개념
* 절대 시간(초)이 아니라 **연산 횟수의 증가 추세**를 보는 것

예시

* 배열 한 번 순회 → n번 반복 → O(n)
* 이중 반복문 → n × n → O(n²)

---

### 2️⃣ 공간 복잡도 (Space Complexity)

* 입력 크기(n)가 증가할 때
* **추가로 사용하는 메모리 공간이 얼마나 필요한지**를 나타내는 개념

예시

* 입력 배열 그대로 사용 → O(1)
* 크기 n짜리 새로운 배열 생성 → O(n)

👉 iOS 개발에서도 중요함
특히 대용량 리스트, 이미지 캐싱, 재귀 호출 시 스택 메모리 등에서 영향 있음

---

### 3️⃣ 빅오(Big-O) 표기법

* 알고리즘의 **최악의 경우 시간/공간 증가율**을 나타내는 표기법
* 상수, 낮은 차수는 무시하고
* 가장 큰 증가율만 표현

예:

```
3n + 5   →  O(n)
2n² + 10n + 3  →  O(n²)
```

👉 왜 상수 무시?
입력 크기가 충분히 커지면 상수는 영향 거의 없음

---

## 4️⃣ O(n) vs O(log n) 차이

### 🔹 O(n)

* 입력 크기만큼 **비례해서 증가**
* n이 100이면 100번
* n이 1,000,000이면 1,000,000번

대표 예:

* 배열 선형 탐색
* 한 번 도는 반복문

---

### 🔹 O(log n)

* 입력이 커질수록 **증가 속도가 매우 느림**
* 보통 "반씩 줄이는 구조"

대표 예:

* 이진 탐색(Binary Search)

예시 비교:

| n         | O(n)      | O(log n) |
| --------- | --------- | -------- |
| 8         | 8         | 3        |
| 1024      | 1024      | 10       |
| 1,000,000 | 1,000,000 | 약 20     |

👉 차이 핵심
O(n)은 선형 증가
O(log n)은 기하급수적으로 줄여가면서 탐색

---

## 📌 면접에서 말하면 좋은 정리 문장

* 시간 복잡도는 입력 크기에 따른 실행 시간 증가율
* 공간 복잡도는 입력 크기에 따른 메모리 사용 증가율
* 빅오는 최악의 경우를 기준으로 상수와 낮은 차수를 제거해 표현
* O(log n)은 O(n)보다 훨씬 효율적인 알고리즘
