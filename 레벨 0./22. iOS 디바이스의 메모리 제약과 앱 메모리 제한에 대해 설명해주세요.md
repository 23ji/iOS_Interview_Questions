#  22. iOS 디바이스의 메모리 제약과 앱 메모리 제한에 대해 설명해주세요.

### iOS 디바이스의 메모리 제약과 앱 메모리 제한

#### **1. 메모리 제약**
- **물리 메모리 한계**: iOS 디바이스는 RAM이 제한적임. 예를 들어, 최신 디바이스도 RAM 용량은 6GB~8GB 수준.
- **앱 메모리 제한**: iOS는 **앱마다 사용할 수 있는 메모리의 상한선**을 설정. 이 한도를 초과하면 앱이 종료됨. 제한은 디바이스 모델에 따라 다름.
- **멀티태스킹 환경**: iOS는 백그라운드 앱과 현재 실행 중인 앱 간에 메모리를 동적으로 할당. 하나의 앱이 과도한 메모리를 사용하면 시스템에서 다른 앱의 메모리를 해제하거나, 심할 경우 해당 앱을 강제로 종료(OOM, Out of Memory)시킴.

---

#### **2. 메모리 경고 (Memory Warning)**
- 메모리가 부족하면 iOS가 앱에 **메모리 경고**를 보냄.
- 개발자는 `UIApplicationDelegate`의 `applicationDidReceiveMemoryWarning` 메서드나 `UIViewController`의 `didReceiveMemoryWarning` 메서드를 활용해 처리할 수 있음.

---

#### **3. 메모리 경고 발생 시 취해야 할 조치**
1. **불필요한 메모리 해제**
   - 더 이상 사용하지 않는 객체나 캐시 데이터를 해제.
   - 이미지, 비디오와 같은 대용량 데이터를 메모리에서 삭제.
   
   ```swift
   override func didReceiveMemoryWarning() {
       super.didReceiveMemoryWarning()
       // 캐시된 이미지 데이터 해제
       imageCache.removeAllObjects()
   }
   ```

2. **Lazy Loading 적용**
   - 필요할 때만 리소스를 로드하여 메모리 사용을 최소화.

3. **백그라운드 작업 종료**
   - 백그라운드에서 실행 중인 작업을 중단하거나 리소스를 정리.

4. **최적화된 데이터 구조 사용**
   - 더 적은 메모리를 사용하는 자료구조로 전환. 예를 들어, 대용량 데이터를 다룰 때 `Array` 대신 스트리밍 방식 사용.

5. **메모리 사용 프로파일링**
   - Xcode의 Instruments 툴에서 메모리 사용을 분석하여 누수(Leak)와 과도한 사용 영역을 파악.

---

#### **4. 메모리 경고를 무시하면?**
- 메모리 경고를 무시하거나 적절히 처리하지 않으면:
  1. 시스템이 강제로 앱을 종료(OOM)시킴.
  2. 사용자 경험 저하 (앱이 느려지거나, 중단).
  3. 백그라운드 앱이 강제 종료되어 시스템 안정성 악화.

---

### 정리
메모리 관리는 iOS 앱 개발에서 필수적인 작업. iOS는 메모리 경고를 통해 앱이 자원을 정리할 기회를 제공하며, 이를 무시하면 앱이 강제 종료될 수 있음. 따라서 메모리 경고 처리 루틴을 구현하고, 메모리 사용을 최적화해야 함.
