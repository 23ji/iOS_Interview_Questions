#  25. 이진 탐색의 원리와 시간 복잡도에 대해 설명해주세요.

## 1️⃣ 핵심 개념

**이진 탐색 (Binary Search)**
→ **정렬된 배열에서 탐색 범위를 절반씩 줄여가며 값을 찾는 알고리즘**

전제 조건

* **데이터가 반드시 정렬되어 있어야 함**

---

## 2️⃣ 동작 원리

과정

1️⃣ 배열 **중간값 선택**
2️⃣ 찾는 값과 **중간값 비교**
3️⃣ 작으면 **왼쪽 탐색**
4️⃣ 크면 **오른쪽 탐색**
5️⃣ 찾을 때까지 반복

예

```
[1, 3, 5, 7, 9]

찾는 값: 7

중간값 = 5
→ 7 > 5

오른쪽 탐색

[7, 9]

중간값 = 7
→ 찾음
```

핵심

**탐색 범위가 매번 절반으로 줄어듦**

---

## 3️⃣ 시간 복잡도

| 상황 | 시간 복잡도       |
| -- | ------------ |
| 평균 | **O(log n)** |
| 최악 | **O(log n)** |

이유

```
n
n/2
n/4
n/8
...
```

→ **log₂ n 단계**

---

## 4️⃣ 선형 탐색과 비교

| 탐색 방식 | 시간 복잡도   |
| ----- | -------- |
| 선형 탐색 | O(n)     |
| 이진 탐색 | O(log n) |

예

데이터 1,000,000개

* 선형 탐색 → 최대 **1,000,000번**
* 이진 탐색 → 약 **20번**

---

## 5️⃣ 면접 답변 요약

이진 탐색은 정렬된 배열에서 중간값을 기준으로 탐색 범위를 절반씩 줄여가며 원하는 값을 찾는 알고리즘입니다. 탐색 범위가 매 단계 절반으로 감소하기 때문에 시간 복잡도는 평균과 최악 모두 O(log n)입니다.

---

## 📌 핵심 요약

* **이진 탐색** → 정렬된 데이터에서 중간값 기준 탐색
* 매 단계 **탐색 범위 절반 감소**
* 시간 복잡도 **O(log n)**
* 정렬되지 않은 데이터에서는 사용 불가
