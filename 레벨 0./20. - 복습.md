#  20. PNG 파일이 어떻게 저장되고 구성되는지 설명해주세요.


## PNG 한 줄 요약

PNG는 **무손실 압축 + 투명도(alpha) 지원**하는 이미지 포맷임

---

## 1️⃣ PNG는 어떻게 저장되나?

PNG는 **바이너리 파일**이고
이미지는 **여러 개의 Chunk(조각)** 로 나뉘어 저장됨

👉 “이미지 하나 = 여러 정보 덩어리들의 묶음”

---

## 2️⃣ PNG 파일 구성 (Chunk 구조)

PNG는 항상 이 순서를 따름

### 필수 Chunk

* **Signature**

  * “이 파일은 PNG임”을 알려주는 표식

* **IHDR**

  * 이미지 기본 정보
  * 가로, 세로, 색상 타입, 비트 깊이

* **IDAT**

  * 실제 픽셀 데이터
  * zlib 기반 **무손실 압축**

* **IEND**

  * 파일 끝 표시

---

### 선택 Chunk

* **PLTE**: 색상 팔레트
* **tRNS**: 투명도 정보
* **gAMA**: 감마 값
* **tEXt**: 텍스트 메타데이터

---

## 3️⃣ PNG 압축 방식 핵심

* **무손실 압축**
* 픽셀 값이 **절대 변하지 않음**
* 필터링 + zlib 압축 사용

👉 확대해도 깨지지 않음
👉 대신 파일 크기 큼

---

## 4️⃣ 투명도(Alpha Channel)

* **RGBA 구조**
* 각 픽셀마다 투명도 값 존재
* 아이콘, UI 이미지에 적합

---

## 5️⃣ PNG vs JPEG 한 줄 비교

* PNG: 무손실, 투명도 가능, 용량 큼
* JPEG: 손실 압축, 투명도 불가, 용량 작음

---

## 📌 평생 기억용 요약

* PNG = **Chunk 단위 구조**
* IHDR → IDAT → IEND 필수
* 무손실 압축 + 투명도 지원
* UI, 아이콘, 로고용 이미지

---

이 문장만 기억하면 됨

> PNG는 “조각(Chunk)으로 나뉜 무손실 + 투명 이미지 파일”
