#  19. iOS에서 이미지 파일 포맷(PNG, JPEG 등)과 각 포맷의 특징에 대해 설명해주세요.


## 한 줄 요약

* **PNG** → 화질 보존 + 투명 배경
* **JPEG** → 용량 작음 + 사진용

---

## 1️⃣ PNG (Portable Network Graphics)

### 특징

* **무손실 압축**
* **투명 배경 지원 (알파 채널)**
* 화질 안 깨짐
* 용량 큼

### 언제 쓰냐?

* 아이콘
* 로고
* 버튼 이미지
* 투명 배경 필요할 때

### 단점

* 사진에는 비효율적 (용량 큼)

---

## 2️⃣ JPEG (JPG)

### 특징

* **손실 압축**
* 투명 배경 ❌
* 용량 작음
* 압축률 조절 가능

### 언제 쓰냐?

* 사진
* 배경 이미지
* 썸네일

### 단점

* 압축하면 화질 손실
* 텍스트/아이콘에는 부적합

---

## 3️⃣ PNG vs JPEG 핵심 비교

| 항목    | PNG    | JPEG    |
| ----- | ------ | ------- |
| 압축 방식 | 무손실    | 손실      |
| 투명 배경 | 가능     | 불가능     |
| 화질    | 유지     | 압축 시 손실 |
| 용량    | 큼      | 작음      |
| 사용처   | UI 리소스 | 사진      |

---

## 4️⃣ iOS 실무 기준 선택법

* **UI 이미지** → PNG
* **사진 데이터** → JPEG
* **네트워크 전송** → JPEG (속도 중요)
* **디자인 정확도 중요** → PNG

---

## 🧠 평생 기억용 문장

> PNG는 **안 깨지고 투명**,
> JPEG는 **가볍고 사진용**
