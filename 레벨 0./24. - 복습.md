#  24. 자주 사용되는 정렬 알고리즘(예: 퀵 정렬, 병합 정렬)의 동작 원리와 시간 복잡도를 설명해주세요.

# 1️⃣ 핵심 개념

정렬 알고리즘
→ 데이터를 **특정 순서로 재배열하는 알고리즘**

대표적으로 면접에서 많이 묻는 것

* **퀵 정렬 (Quick Sort)**
* **병합 정렬 (Merge Sort)**

둘 다 **분할 정복(Divide and Conquer)** 기반

---

# 2️⃣ 퀵 정렬 (Quick Sort)

### 동작 원리

1️⃣ **Pivot(기준값)** 선택
2️⃣ Pivot 기준으로 **작은 값 / 큰 값 분리**
3️⃣ 각 그룹에 대해 **재귀적으로 동일 과정 반복**

구조

```
배열
 ↓
pivot 기준 분할
 ↓
왼쪽 / 오른쪽 재귀 정렬
```

### 특징

* **평균적으로 매우 빠름**
* **In-place 정렬 가능**
* pivot 선택에 따라 성능 영향 있음

### 시간 복잡도

| 상황 | 시간 복잡도     |
| -- | ---------- |
| 평균 | O(n log n) |
| 최악 | O(n²)      |

최악 상황
→ 이미 정렬된 배열 + pivot 선택이 나쁠 때

---

# 3️⃣ 병합 정렬 (Merge Sort)

### 동작 원리

1️⃣ 배열을 **계속 반으로 분할**
2️⃣ **단일 요소까지 분해**
3️⃣ **정렬하면서 병합**

구조

```
Divide
배열 반으로 분할

↓

Sort
각 부분 정렬

↓

Merge
두 배열을 비교하며 합침
```

### 특징

* **항상 안정적인 성능**
* **Stable Sort**
* **추가 메모리 필요**

### 시간 복잡도

| 상황 | 시간 복잡도     |
| -- | ---------- |
| 평균 | O(n log n) |
| 최악 | O(n log n) |

---

# 4️⃣ 퀵 정렬 vs 병합 정렬

| 항목     | 퀵 정렬  | 병합 정렬      |
| ------ | ----- | ---------- |
| 평균 성능  | 빠름    | 안정적        |
| 최악 성능  | O(n²) | O(n log n) |
| 추가 메모리 | 거의 없음 | 필요         |
| 정렬 안정성 | 불안정   | 안정         |

---

# 5️⃣ 면접 한 줄 답변 예시

퀵 정렬은 Pivot을 기준으로 배열을 분할하며 정렬하는 알고리즘으로 평균 시간 복잡도는 O(n log n)이고 최악의 경우 O(n²)입니다.
병합 정렬은 배열을 계속 반으로 분할한 뒤 정렬하면서 병합하는 알고리즘이며 항상 O(n log n)의 시간 복잡도를 보장하지만 추가 메모리가 필요합니다.

---

# 📌 핵심 요약

* **퀵 정렬** → Pivot 기준 분할 정렬
* 평균 **O(n log n)** / 최악 **O(n²)**
* **병합 정렬** → 분할 후 병합 정렬
* 항상 **O(n log n)** 보장
* 퀵 정렬은 빠르고, 병합 정렬은 안정적
