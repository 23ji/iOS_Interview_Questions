#  15. Swift의 에러 처리 방법에 대해 설명해주세요.

Swift에서 에러 처리는 안전하게 오류를 다루기 위해 제공되는 구조임

---

### 1️⃣ 기본 개념

* **에러(Error)**: 함수 실행 중 문제가 발생했음을 나타내는 값
* Swift에서는 `Error` 프로토콜을 채택한 타입을 에러로 정의할 수 있음

```swift
enum MyError: Error {
    case fileNotFound
    case invalidInput
}
```

---

### 2️⃣ `throws` / `try` / `catch` 사용법

* **`throws`**: 함수가 에러를 던질 수 있음을 표시
* **`try`**: 에러를 던지는 함수 호출 시 사용
* **`do-catch`**: 에러가 발생했을 때 처리

```swift
func readFile(name: String) throws -> String {
    if name.isEmpty { throw MyError.fileNotFound }
    return "File Content"
}

do {
    let content = try readFile(name: "")
    print(content)
} catch MyError.fileNotFound {
    print("파일을 찾을 수 없음")
} catch {
    print("알 수 없는 에러")
}
```

* `do` 블록 안에서 `try` 사용 → `catch` 블록에서 에러 처리

---

### 3️⃣ 옵셔널과의 차이

| 구분 | 옵셔널           | do-catch           |
| -- | ------------- | ------------------ |
| 동작 | 실패 시 `nil` 반환 | 실패 시 에러 던지고 처리     |
| 장점 | 간단한 실패 처리     | 실패 이유를 구체적으로 확인 가능 |
| 단점 | 실패 이유 알기 어려움  | 코드가 길어질 수 있음       |

```swift
// 옵셔널 예
func readFileOptional(name: String) -> String? {
    return name.isEmpty ? nil : "File Content"
}

let content = readFileOptional(name: "")
print(content ?? "파일 없음") // 파일 없음
```

---

### 4️⃣ 에러 전파

* 함수에서 직접 처리하지 않고 호출자에게 에러 전달 가능
* `throws`를 사용하면 에러를 **상위 함수로 전파**

```swift
func processFile() throws {
    try readFile(name: "")
}

do {
    try processFile()
} catch {
    print("에러 처리")
}
```

* 여러 단계 함수에서 한 번만 처리하면 됨 → 코드 간결

---

### 5️⃣ 정리

* `throws` → 에러 발생 가능 표시
* `try` → 에러 던지는 함수 호출
* `do-catch` → 에러 처리
* 옵셔널은 실패 이유를 알 수 없고, do-catch는 구체적 에러 처리 가능
* 에러를 상위로 전파하여 중앙에서 관리 가능
