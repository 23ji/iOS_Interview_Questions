#  25. 동시성 프로그래밍에서 동기(Synchronous)와 비동기(Asynchronous)의 차이점은 무엇인가요?

# 1️⃣ 동기(Synchronous)와 비동기(Asynchronous) 개념

## 동기 (Synchronous)

**작업이 끝날 때까지 다음 작업이 기다림**

즉

```id="s4n2s8"
1번 작업 끝남
↓
2번 작업 시작
↓
3번 작업 시작
```

앞 작업이 끝나야 다음 진행 가능

---

## 비동기 (Asynchronous)

**작업 시작만 시키고 끝날 때까지 기다리지 않음**

즉

```id="fy5rk0"
1번 작업 시작
↓
기다리지 않고 2번 작업 시작
↓
1번 끝나면 나중에 결과 처리
```

---

# 2️⃣ 왜 비동기가 필요할까

iOS에서 가장 중요함

이유

메인 스레드는 UI 담당

여기서 오래 걸리는 작업 하면

```id="qkh1ey"
화면 멈춤
터치 안됨
스크롤 끊김
```

대표 예

* 네트워크 요청
* 파일 읽기
* 이미지 다운로드
* DB 처리

그래서 비동기로 돌림

---

# 3️⃣ 실생활 비유

## 동기

카페 주문

```id="h0h84t"
커피 다 나올 때까지 서 있음
```

다음 일 못함

## 비동기

```id="bq2r0f"

벨 울리면 찾으러 감
```

iOS 비동기가 이 구조

많이 헷갈리는 부분


# 5️⃣ iOS에서 비동기 처리 방법

대표 4개

* DispatchQueue

# 6️⃣ DispatchQueue

가장 기본

```swift
// File: NetworkManager.swift
// Layer: Network

DispatchQueue.global().async {
    
    let data = heavyTask()
    
    DispatchQueue.main.async {
        updateUI(data)
    }
}
```

---

## 구조

```id="h3rj0w"
global()
→ 백그라운드 작업

main
→ UI 업데이트
```

---

## 왜 main으로 다시 와야 하나

UIKit / SwiftUI는 메인 스레드 전용

즉

```id="4t0r8f"
UI 수정 = main queue
```

필수

---

# 7️⃣ Completion Handler

비동기 완료 후 결과 전달

```swift
// File: APIService.swift
// Layer: Network

func fetchUser(completion: @escaping (String) -> Void) {
    
    DispatchQueue.global().async {
        completion("User Loaded")
    }
}
```

사용

```swift
fetchUser { result in
    print(result)
}
```

---

# 8️⃣ async / await

최신 Swift 방식

가독성 매우 좋음

```swift
// File: APIService.swift
// Layer: Network

func fetchUser() async -> String {
    return "User Loaded"
}
```

사용

```swift
Task {
    let user = await fetchUser()
    print(user)
}
```

# 9️⃣ Semaphore

## 개념

**동시에 접근 가능한 개수 제한**

예

리소스 3개 있음

→ 3개까지만 접근 허용

---

## 비유

주차장 자리 3개

```id="45n97f"
자리 있으면 입장
자리 없으면 대기
```

---

## 코드

```swift
// File: SemaphoreExample.swift
// Layer: Concurrency

let semaphore = DispatchSemaphore(value: 2)

semaphore.wait()

// 작업

semaphore.signal()
```

---

## 핵심

```id="3m2eqh"
count 기반 제어
```

---

# 🔟 Mutex

## 개념

**오직 1개 스레드만 접근 허용**

---

## 비유

화장실 1칸 🚪

```id="w8q5k4"
사용 중이면 기다림
```

---

## 핵심

공유 자원 보호

---

# 1️⃣1️⃣ Semaphore vs Mutex 차이

| 항목    | Semaphore | Mutex    |
| ----- | --------- | -------- |
| 허용 개수 | 여러 개 가능   | 1개만 가능   |
| 관리 방식 | count     | lock     |
| 목적    | 자원 수 제한   | 임계 영역 보호 |

---

## 쉽게 기억

```id="r5k7y2"
Semaphore = 몇 명까지 들어올지 제한

Mutex = 한 명만 들어감
```

---

# 1️⃣2️⃣ 면접 답변 요약

동기는 작업이 끝날 때까지 다음 작업이 대기하는 방식이고, 비동기는 작업 완료를 기다리지 않고 다음 작업을 수행하는 방식입니다. iOS에서는 DispatchQueue, Completion Handler, async/await 등을 사용해 비동기 작업을 처리합니다. Semaphore는 접근 가능한 자원 수를 count로 제어하고, Mutex는 한 번에 하나의 스레드만 임계 영역에 접근하도록 lock을 사용합니다.

---

# 📌 핵심 요약

* **동기** → 끝날 때까지 기다림
* **비동기** → 시작만 하고 다음 진행
* iOS 비동기 핵심 → `DispatchQueue`, `async/await`
* **Semaphore** → count 기반 제한
* **Mutex** → lock 기반 단일 접근
